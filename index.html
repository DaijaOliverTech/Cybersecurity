Project: Cybersecurity Incident Response and Web Application Vulnerability Assessment

Project Overview

This project simulates a realistic cybersecurity scenario involving a web application vulnerability assessment, detection of an attack, and coordinated incident response. It demonstrates technical and analytical skills across the full incident lifecycle.

Tools Used

  • OWASP Juice Shop (vulnerable test application)
  • OWASP ZAP (vulnerability scanner)
  • Wireshark (network packet analysis)
  • Burp Suite Community Edition (manual testing)
  • Kali Linux (penetration testing environment)
  • ServiceNow / JIRA (simulated ticket tracking)

Methodology

1. Preparation

  • Created a virtual lab with Juice Shop deployed on localhost.
  • Reviewed OWASP Top 10 to understand common web vulnerabilities.
  • Defined incident response roles and documentation workflow.

2. Vulnerability Assessment

  • Scanned Juice Shop using OWASP ZAP.
  • Discovered vulnerabilities including XSS, SQL Injection, and improper authentication.
  • Prioritized findings using CVSS scores and created a risk matrix.

3. Detection and Analysis

  • Simulated a credential stuffing attack based on discovered vulnerabilities.
  • Monitored logs and network traffic using Wireshark.
  • Documented IOC (Indicators of Compromise) and attack timeline.

4. Containment, Eradication, and Recovery

  • Contained the incident by disabling affected services.
  • Patched identified vulnerabilities and reconfigured authentication mechanisms.
  • Validated fixes through rescanning and log review.

5. Post-Incident Activity

  • Conducted a lessons-learned meeting (documented).
  • Proposed updated WAF rules, regular scanning schedules, and team training.
  • Documented and archived the entire incident lifecycle in a simulated ServiceNow instance.

Deliverables

  • Vulnerability scan report (OWASP ZAP)
  • Incident response timeline and logs
  • IOC list and network traffic captures
  • Remediation plan
  • Lessons-learned report

Skills Demonstrated

  • Web vulnerability scanning and triage
  • Incident detection and forensic analysis
  • Communication of technical findings
  • Risk-based prioritization
  • Use of cybersecurity frameworks (NIST SP 800-61r2, OWASP WSTG)
